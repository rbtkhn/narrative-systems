# Lecture Packet Template

Copy all five files for a completed lecture. Replace bracketed fields and keep
the artifacts non-duplicative.

## Reading Order

1. `manuscript.md` — canonical public lecture.
2. `commentary.md` — interpretation, counter-readings, and value review.
3. `sources.md` — evidence and candidate-conversion record.
4. `orientation.yaml` — machine-readable routing and state.

`orientation.yaml` is authoritative for mechanical status and paths. Status
language elsewhere in the packet is explanatory and must follow it.

## Doorway Fields

- ID and title
- volume position
- status and public-readiness state
- 3–5 learning outcomes
- guardrails and scope boundary
- canonical-manuscript rule

Status vocabulary: `brief`, `draft`, `source-review`, `reviewed`, `public`.
Public readiness is a separate boolean; `reviewed` does not automatically mean
published.
